100 Days of Java with Ankitha

🎯 Objective

Learn how the Java Virtual Machine (JVM) executes Java programs and manages memory.

📖 What is JVM?

JVM (Java Virtual Machine) is a part of the Java Runtime Environment (JRE). It converts Java bytecode into machine code, making Java platform independent.

🌟 Why JVM?

  • Platform Independent
  • Automatic Memory Management
  • Garbage Collection
  • Security
  • Improved Performance using JIT Compiler

🏗️ JVM Architecture

  1. Class Loader Subsystem
  2. Runtime Data Areas
  3. Execution Engine
  4. Java Native Interface (JNI)
  5. Native Method Libraries

📦 Runtime Data Areas

Memory Area Purpose
Heap Stores Objects and Instance Variables
Stack Stores Local Variables and Method Calls
Method Area Stores Class Metadata and Static Variables
PC Register Stores Current Instruction Address
Native Method Stack Stores Native Methods

⚙️ Java Program Execution Flow

Bytecode (.class)
        ↓
Class Loader
        ↓

        ↓
Execution Engine
        ↓

        ↓

        ↓
Output
