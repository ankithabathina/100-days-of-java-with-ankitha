📖 What is Multithreading?

Multithreading is a feature of Java that allows multiple threads to run concurrently within a single process. Each thread performs an independent task while sharing the same memory space of the process. Instead of executing one task after another, multiple tasks can execute simultaneously, improving application performance and responsiveness.

❓ Why Do We Need Multithreading?

  • Improves application performance.
  • Better CPU utilization.
  • Executes multiple tasks simultaneously.
  • Supports background processing.
  • Makes applications more responsive.
  • Efficient resource sharing.
  • Reduces overall execution time.

📝 Internal Working Explanation

Step 1: The JVM starts executing the program with the Main Thread.

Step 2: A new Thread object is created using the Thread class or Runnable interface.

Step 3: Calling start() requests the JVM to create a new thread.

Step 4: The JVM internally invokes the run() method.

Step 5: The Main Thread and Child Thread execute concurrently.

Step 6: Once execution is completed, the thread enters the TERMINATED state.

📌 Thread States Explanation

1️⃣ NEW

A thread is in the NEW state when it is created but the start() method has not yet been called.

2️⃣ RUNNABLE

After calling start(), the thread enters the RUNNABLE state and waits for CPU scheduling.

3️⃣ RUNNING

When the CPU allocates time to the thread, it moves to the RUNNING state and executes the run() method.

4️⃣ WAITING

A thread enters the WAITING state when it waits for another thread to complete or for a specific event to occur.

5️⃣ BLOCKED

A thread enters the BLOCKED state when it is waiting to acquire a lock or monitor before continuing execution.

6️⃣ TERMINATED

Once the thread completes execution or stops due to an exception, it enters the TERMINATED state.

🔑 Ways to Create a Thread

1️⃣ Extending Thread Class

A class can create a thread by extending the Thread class and overriding its run() method. 

class MyThread extends Thread {

    public void run() {
        System.out.println("Thread is Running");
    }

}

Advantage: Easy to implement for simple applications.

Limitation: Since Java supports only single inheritance, the class cannot extend any other class.

2️⃣ Implementing Runnable Interface

Another way to create a thread is by implementing the Runnable interface and overriding the run() method. 

class MyRunnable implements Runnable {

    public void run() {
        System.out.println("Runnable is Running");
    }

}

Advantage: Allows the class to extend another class and is the preferred approach in Java.

⭐ Important Thread Methods

Method Description
start() Starts a new thread.
run() Contains the code executed by the thread.
sleep() Pauses the current thread for a specified time.
join() Waits until another thread finishes execution.
yield() Temporarily gives another thread a chance to execute.
interrupt() Interrupts a sleeping or waiting thread.
isAlive() Checks whether a thread is currently running.
currentThread() Returns the currently executing thread.

✅ Advantages of Multithreading

  • Improves application performance.
  • Better CPU utilization.
  • Supports concurrent execution.
  • Makes applications more responsive.
  • Efficient resource sharing.
  • Background task execution.
  • Reduces overall execution time.

⚠️ Disadvantages of Multithreading

  • Difficult debugging.
  • Thread synchronization issues.
  • Deadlock can occur.
  • Race conditions may occur.
  • Program complexity increases.

💡 Best Practices

  • Prefer Runnable Interface over extending Thread.
  • Avoid creating unnecessary threads.
  • Use synchronization for shared resources.
  • Always handle InterruptedException.
  • Use Thread Pools (Executor Framework) in large applications.
  • Keep thread execution lightweight.
  • Avoid deadlocks by acquiring locks carefully.

📌 Key Takeaways

  • Multithreading allows multiple threads to execute concurrently.
  • Threads share the memory of the same process.
  • Java provides Thread Class and Runnable Interface for thread creation.
  • Understanding the Thread Lifecycle is essential for writing concurrent applications.
  • Synchronization helps prevent data inconsistency when multiple threads access shared resources.
  • Multithreading improves performance but should be used carefully to avoid deadlocks and race conditions.
