📖 What is Synchronization?

Synchronization is a mechanism in Java that allows only one thread to access a shared resource at a time. Whenever multiple threads try to modify the same object, Java uses synchronization to prevent data corruption and maintain consistency. Synchronization is achieved using object locks (monitors). Before entering a synchronized method or block, a thread must first acquire the object's lock.

❓ Why Do We Need Synchronization?

Imagine two ATM machines trying to withdraw money from the same bank account simultaneously. If both machines access the balance at the same time, the final balance may become incorrect. Synchronization prevents such problems by allowing only one thread to perform the operation at a time.

⚠️ What is a Race Condition?

A Race Condition occurs when two or more threads access and modify shared data simultaneously, resulting in unpredictable or incorrect output. Example: 


Initial Count = 0

Thread A reads 0

Thread B reads 0

Thread A increments → 1

Thread B increments → 1

Final Count = 1

Expected Count = 2

This problem occurs because both threads accessed the shared variable without synchronization.

Whenever a thread enters a synchronized method, the JVM checks whether the object's lock is available. If available, the thread acquires the lock and executes. All remaining threads wait until the lock is released.
