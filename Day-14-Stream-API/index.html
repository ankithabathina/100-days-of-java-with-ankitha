📖 What is Stream API?

Stream API was introduced in Java 8. It provides a functional approach for processing collections of data. A Stream is not a data structure. It is a sequence of elements that supports various operations such as filtering, mapping, sorting, searching, and collecting data. Unlike collections, Streams do not store data. They simply process data.

❓ Why Stream API?

Before Java 8, developers had to write lengthy loops to process collections. Stream API reduces boilerplate code and makes programs cleaner, shorter, and easier to understand.

Traditional Approach Stream API

A Stream Pipeline consists of three stages: 1. Source (Collection) 2. Intermediate Operations 3. Terminal Operation The stream processes data step by step until the final result is produced.

🔹 Intermediate Operations

Intermediate operations return another Stream. They are lazy, meaning they execute only when a terminal operation is called.

Method Purpose
filter() Select elements
map() Transform elements
sorted() Sort elements
distinct() Remove duplicates
limit() Limit elements
skip() Skip elements
peek() Debug Stream

🔹 Terminal Operations

Terminal operations produce the final result and close the stream.

Method Purpose
collect() Collect result
forEach() Print elements
count() Count elements
reduce() Reduce to single value
findFirst() Return first element
findAny() Return any element
max() Maximum value
min() Minimum value
