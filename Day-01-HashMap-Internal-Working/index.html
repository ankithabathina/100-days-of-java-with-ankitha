📘 Day 1 - How HashMap Works Internally in Java

🎯 Objective

Learn how HashMap stores and retrieves data efficiently using hashing.

📖 What is HashMap?

HashMap is a class in the Java Collections Framework that stores data as key-value pairs. It uses hashing to provide fast insertion, deletion, and retrieval of elements.

⚙️ Internal Working

  1. Calculate the hash code of the key.
  2. Convert the hash code into a bucket index.
  3. Store the key-value pair in that bucket.
  4. Handle collisions using Linked List or Red-Black Tree.

💻 Example

HashMap<Integer, String> map = new HashMap<>();

map.put(1, "Java");
map.put(2, "Spring Boot");
map.put(3, "React");

⏱️ Time Complexity

Operation Complexity
Insert O(1)
Search O(1)
Delete O(1)

Worst Case: O(n)

📌 Key Takeaways

  • Stores data as key-value pairs.
  • Uses hashing for fast retrieval.
  • Allows one null key and multiple null values.
  • Does not maintain insertion order.
  • Handles collisions using Linked List or Red-Black Tree.
