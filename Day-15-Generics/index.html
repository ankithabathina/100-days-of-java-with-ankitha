📖 What are Generics?

Generics are a feature introduced in Java 5. They allow classes, interfaces, and methods to work with different data types while maintaining compile-time type safety. Instead of writing separate code for Integer, String, Double, or Employee objects, Generics let us write one reusable implementation.

❓ Why were Generics introduced?

Before Java 5, collections stored everything as an Object. Whenever we retrieved data, we had to manually convert it back to its original type. This process is called Type Casting. If we cast the wrong type, Java throws a ClassCastException. Generics solve this problem by checking the data type during compilation. 


Without Generics

ArrayList list = new ArrayList();

list.add("Java");

list.add(100);

String s = (String) list.get(0);

Problems ✔ No Type Safety ✔ Manual Type Casting ✔ Runtime Errors

✅ With Generics


ArrayList<String> list =
new ArrayList<>();

list.add("Java");

// list.add(100);

String s = list.get(0);

Benefits ✔ Type Safety ✔ No Explicit Type Casting ✔ Compile-Time Checking ✔ Better Readability

🔹 Generic Type Parameters

Generics use angle brackets (<>) to specify the type of object. Common naming conventions are:

Type Meaning
T Type
E Element
K Key
V Value
N Number

Example 


HashMap<Integer,String>

Integer → Key String → Value

🏗 Generic Class

A Generic Class allows us to create objects of different data types using the same class. Instead of creating multiple classes, one Generic Class can handle many object types.

Syntax


class Box<T>{

}

Example


class Box<T>{

T value;

void set(T value){

this.value=value;

}

T get(){

return value;

}

}

Here, T represents a data type. When we create Box<String> T becomes String.

⚙ Generic Method

A Generic Method works with multiple data types. Instead of creating different methods for Integer, String, Double, etc., one Generic Method handles all object types.

Syntax


public static <T>
void print(T value)

Example


print(100);

print("Java");

print(25.5);

🖥 Generic Interface

Interfaces can also use Generics. The implementing class decides the data type. 


interface Printer<T>{

void print(T value);

}

Example 


class StringPrinter

implements Printer<String>{
