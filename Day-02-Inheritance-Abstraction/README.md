# Day 2 – Inheritance, Abstraction, Interface & Abstract Class

## 📖 Overview

Welcome to **Day 2** of my **#100DaysOfJava** journey.

Today, I learned the core Object-Oriented Programming (OOP) concepts that help in writing reusable, maintainable, and scalable Java applications.

### 📚 Topics Covered

* Inheritance
* Abstraction
* Interface
* Abstract Class
* Concrete Class
* Full Abstraction vs Partial Abstraction
* Advantages and Disadvantages of Inheritance

### 📂 Files Included

* **Inheritance.md** – Concepts, types, advantages, disadvantages, and examples.
* **Abstraction.md** – Definition, real-world examples, and types of abstraction.
* **Interface.md** – Features, methods, variables, and examples.
* **AbstractClass.md** – Features, abstract methods, concrete methods, and examples.
* **InheritanceDemo.java** – Java program demonstrating inheritance.
* **InterfaceDemo.java** – Java program demonstrating interfaces.
* **AbstractClassDemo.java** – Java program demonstrating abstract classes.

### 🎯 Learning Outcome

By the end of Day 2, I understood:

* How inheritance enables code reusability.
* Why abstraction hides implementation details.
* The difference between an interface, an abstract class, and a concrete class.
* When to use an interface and when to use an abstract class.
* The difference between full abstraction and partial abstraction.
